An LXC has no kernel of its own, so /var/run/reboot-required almost never appears — pick “always” if you want the container restarted after each update anyway. A container that fails to come back is rolled back to the pre-update snapshot.

Some app updates build from source (npm install, compiles) and run out of memory on a small container — the update fails with rc=137 (OOM) or rc=113 (under-provisioned). adminupdater raises this container's RAM just for the update and puts it back afterwards; the Proxmox host caps the value (ram_boost_max_mb in host.conf).

A stopped container is normally skipped (its snapshot and its packages stay as they are). “Start it” powers it on, waits until it really answers, updates it, and then either shuts it down again or leaves it up.
